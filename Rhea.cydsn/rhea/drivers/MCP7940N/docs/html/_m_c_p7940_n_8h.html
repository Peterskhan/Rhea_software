◆ MCP7940N_ReadRegisters()

void MCP7940N_ReadRegisters ( void  )

Updates the local copy of the register structure from the physical device.

The library functions are working on a local copy of the device registers stored in memory. To get the latest data out of the device MCP7940N_ReadRegisters() must be called. This is necessary for data consistency and efficiency reasons, it is faster to read all of the registers of the device at the same time then reading them individually, and data consistency is also ensured, as the visible registers of the device are frozen during a sequential read operation.

◆ MCP7940N_SetI2CReader()

void MCP7940N_SetI2CReader ( MCP7940N_I2CReaderFn  reader)

Sets the I2C reader interface function for the library. The library will read from the I2C bus using the function pointer provided.

Parameters
[in]readerA pointer to the function that facilitates the sequential read on the I2C bus.

The provided function pointer must reference a function that facilitates a sequential read on the specific I2C bus. First, the device has to be selected by sending the control byte after the START condition. The last bit of the control byte must be 0, to indicate the write operation. The device acknowledges the call, then expects a register/SRAM address which is then set internally. After the register/SRAM address is sent a repeated START is generated after the ACK. The process is show below for reading one byte (from the official datasheet):

Reading one byte from the MCP7940N

After setting the device's internal address register, the control byte is issued again, which the device will acknowledge. This time the last bit of the control byte must be 1, to indicate read operation. The device then transmits the first data byte, which the reader must acknowledge. The device will continue to send the data bytes until the last one is indicated by sending a NACK and a STOP condition. The process is show below for reading multiple bytes after the address has been set (from the official datasheet):

void MCP7940N_SetI2CWriter ( MCP7940N_I2CWriterFn  writer)

Sets the I2C writer interface function for the library. The library will write to the I2C bus using the function pointer provided.

Parameters
[in]writerA pointer to the function that facilitates the sequential write on the I2C bus. See details below.

The provided function pointer must reference a function that facilitates a sequential write on the specific I2C bus. First, the device has to be selected by sending the control byte after the START condition. The last bit of the control byte must be 0, to indicate the write operation. The device acknowledges the call, then expects a register/SRAM address which is then set internally. After the register/SRAM address is sent and acknowledged, the data bytes can be written sequentially, with each being acknowledged by the device. ◆ MCP7940N_WriteRegisters()

void MCP7940N_WriteRegisters ( void  )

Updates the registers of the physical device from the local copy stored in memory.

The library functions are working on a local copy of the device registers stored in memory. To reflect any changes in the local register values, the device must be updated via the I2C bus. This is necessary for efficiency and safety reasons, it is faster to write all of the registers of the device at the same time then writing them individually, and safety is also ensured by avoiding data corruption.
