uint8_t BME280_GetHumidity ( void  )

Returns the relative humidity in percents.

For humidity compensation by temperature to be fully accurate, either BME280_GetTemperatureCelsius() or BME280_GetTemperatureFahrenheit() must be called prior to this function. The humidity readout and compensation also operates from the local registers, so BME280_ReadRegisters() must be called before calling this function.

The relative humidity in percents.

◆ BME280_GetTemperatureCelsius()

int8_t BME280_GetTemperatureCelsius ( void  )

Returns the temperature in degrees Celsius.

This function saves an intermediate version of the measured temperature used in compensation of pressure and humidity measurements. For those measurements to be fully accurate, BME280_GetTemperatureCelsius() or BME280_GetTemperatureFahrenheit() must be called first. This function calculates from the locally stored register values, so BME280_ReadRegisters() must be called before calling this function.

Returns
The temperature in degrees Celsius.

◆ BME280_ReadRegisters()

void BME280_ReadRegisters ( void  )

Updates the measurement registers of the local copy from the physical device.

Most library functions are working on a local copy of the device registers stored in memory. To get the latest data out of the device BME280_ReadRegisters() must be called. This is necessary for data consistency and efficiency reasons, it is faster to read all of the registers of the device at the same time then reading them individually, and data consistency is also ensured, as the visible registers of the device are frozen during a sequential read operation.

◆ BME280_SetI2CReader()

void BME280_SetI2CReader ( BME280_I2CReaderFn  reader)

Sets the I2C reader interface function for the library. The library will read from the I2C bus using the function pointer provided.

Parameters
[in]readerA pointer to the function that facilitates the sequential read on the I2C bus.

The provided function pointer must reference a function that facilitates a sequential read on the specific I2C bus. First the device has to be selected by sending the I2C address byte in write mode after the START condition. The device acknowledges, then expects the control byte which is the first register address being read. After that a repeated START condition is generated and the device is addressed again, this time in read mode. Following the acknowledge the device starts sending the auto-incremented register contents, which the master must acknowledge. The transfer is continued until the last byte transaction is followed by a NACK and a STOP condition.

◆ BME280_SetI2CWriter()

void BME280_SetI2CWriter ( BME280_I2CWriterFn  writer)

Sets the I2C writer interface function for the library. The library will write to the I2C bus using the function pointer provided.

Parameters
[in]writerA pointer to the function that facilitates the sequential write on the I2C bus. See details below.

The provided function pointer must reference a function that facilitates a sequential write on the specific I2C bus. First the device has to be selected by sending the I2C address byte in write mode after the START condition. The device acknowledges, then expects the control byte which is the first register address being written. Following the acknowledge the data byte to write is sent. The next bytes can be written by sending the register address first, then sending the data to be written. The transfer continues until the last byte transaction is followed by a STOP condition. For the write operation, register addresses are not auto-incremented, so the user must send address & data byte pairs in this order.

◆ BME280_WriteRegisters()

void BME280_WriteRegisters ( void  )

Updates the configuration registers of the physical device from the local copy stored in memory.

Most library functions are working on a local copy of the device registers stored in memory. To reflect any changes in the local register values, the device must be updated via the I2C bus. This is necessary for efficiency and safety reasons, it is faster to write all of the registers of the device at the same time then writing them individually, and safety is also ensured by avoiding data corruption.
